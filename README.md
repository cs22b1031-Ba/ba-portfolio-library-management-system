# Library Management System

This portfolio project presents a Business Requirements Document package for a **Library Management System Implementation** in an Indian college or public library context. It was created to showcase practical Business Analyst skills in a way that feels close to a real client-facing assignment rather than a classroom exercise. The project covers core BA deliverables such as scope definition, stakeholder analysis, process mapping, functional requirements, non-functional requirements, user stories, and solution visualization.

## Portfolio Objective

The objective of this project is to show how a Business Analyst can take an operational problem, understand the current process, identify pain points, and translate that understanding into a structured requirement package that is clear, useful, and presentation-ready for stakeholders.

## Deliverables

- `BRD_Library_System.md`: Formal BRD covering business context, objectives, scope, stakeholders, requirements, target process, and success metrics.
- `AS-IS_Process_Flow.svg`: Professionally styled current-state process flow highlighting manual steps and pain points.
- `TO-BE_Process_Flow.svg`: Future-state process flow showing the proposed automated workflow and business outcomes.
- `Wireframes.svg`: Clean wireframes for key LMS screens to support requirement visualization and stakeholder discussion.
- `User_Stories.xlsx`: User story backlog with story statements, priority, and acceptance criteria.

## Problem Statement

Many libraries still depend on registers, handwritten issue logs, and spreadsheets for circulation and inventory tracking. While these methods may work at a basic level, they often create slow service, inconsistent records, limited visibility into stock, weak reporting, and avoidable frustration for both staff and members. This project proposes a Library Management System that addresses those gaps through a centralized, policy-driven platform for catalog management, circulation, reservations, fines, and reporting.

## BA Deliverables Demonstrated

- Business requirements documentation
- Scope definition and boundary setting
- Stakeholder analysis
- As-is and to-be process mapping
- Functional and non-functional requirement definition
- User story creation
- Wireframe-based solution visualization

## Key Functional Areas

- Member management
- Book catalog and copy tracking
- Issue, return, renewal, and reservation workflows
- Fine calculation and overdue monitoring
- Notifications and communication
- Dashboards, reports, and audit support


This artifact set is structured to resemble a client-facing BA submission, with realistic content, a clear problem statement, structured requirements, supporting visuals, and measurable business outcomes. It is intended to support internship applications, interviews, case study walkthroughs, and portfolio discussions by demonstrating both analysis thinking and presentation quality.
